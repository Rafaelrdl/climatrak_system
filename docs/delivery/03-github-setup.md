## Como usar
1) Crie labels no repositório
2) Crie milestones M0..M5
3) Crie issues copiando de `docs/delivery/02-backlog-issues.md`
4) Atribua milestone e labels em cada issue
5) (Opcional) Crie um GitHub Project (Kanban):
   - Backlog
   - Ready
   - In Progress
   - Review
   - Done
